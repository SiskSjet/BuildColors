This mod allows you to create customized color sets for building, which can then be easily imported into another world.

[h2]🛠︰Color Sets ・────────・・・[/h2]
To save or load a color set you have to type the listed commands below in your chat box.

[CODE]Usage: /bc [command] [arguments][/CODE]

[b]Available commands[/b]:
[table]
[tr]
	[th]Command[/th]
	[th]Arguments[/th]
	[th]Description[/th]
[/tr]
[tr]
	[td]save[/td]
	[td]name[/td]
	[td]Saves a Color Set with the given name.[/td]
[/tr]
[tr]
	[td]load[/td]
	[td]name[/td]
	[td]Loads a Color Set with the given name.[/td]
[/tr]
[tr]
	[td]remove[/td]
	[td]name[/td]
	[td]Removes a Color Set with given name.[/td]
[/tr]
[tr]
	[td]generate[/td]
	[td][/td]
	[td]generates a random color set. It is not saved, only the color palette is changed[/td]
[/tr]
[tr]
	[td]list[/td]
	[td][/td]
	[td]Lists all available color sets.[/td]
[/tr]
[tr]
	[td]help[/td]
	[td][/td]
	[td]Shows a help window with all commands[/td]
[/tr]
[/table]

[h2]🛠︰Configs ・──────────・・・[/h2]
This mod can create two config files.
[OLD can be removed] [strike]For servers a settings.xml file is created in your world storage.[/strike]
Color sets are saved in ColorSets.xml in your global storage:
"%appdata%\SpaceEngineers\Storage\ColorSets.xml"

[h2]🛠︰Known issues ・──────・・・[/h2]
The UI has a problem when naming a color set. Pressing keys bound to game actions triggers those actions instead. For example, pressing 'P' closes the Color-picker Screen. While it's a bit annoying, pressing 'P' again allows you to continue. Unfortunately, I don't know how to fix this issue.

[h2]🛠︰BUGS ・─────────────・・・[/h2]
Es gibt ein Problem mit der Benutzeroberfläche, wenn du versuchst, einen Farbsatz zu benennen. Wenn du Tasten drückst, die an Spielfunktionen gebunden sind, wird die Spielfunktion ausgeführt. Wenn du beispielsweise 'P' drückst, wird der Farbauswahlbildschirm geschlossen. Es ist etwas nervig, aber du kannst erneut 'P' drücken, um fortzufahren. Leider habe ich keine Ahnung, wie man das beheben kann.
